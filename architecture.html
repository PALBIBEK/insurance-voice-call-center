Persistence
Postgres Durable record: sessions, turns, handoffs, tool attempts, claims, approvals, trace spans. (SQLite in tests.)
Redis Hot session state + pub/sub fan-out across workers. In-memory fallback for single-process dev.
External LLM
OpenRouter Model inference via AsyncOpenAI client. Absent a key, the demo brain replaces it — same agents, tools and events.

How to read this

Voice path Caller speaks → ElevenLabs transcribes → webhook POSTs the utterance to /api/voice/completions → the brain runs one agent turn → tokens stream back over SSE → ElevenLabs speaks them.
Console path The same turn's events (agent handoffs, tool attempts, approval requests, drift flags) publish through the event bus to the WebSocket, so the console mirrors the call live and can approve/reject claims.
One turn Gateway pipeline is strict: guardrails first, then the agent loop (which may call tools / hand off), then persist to Postgres before publishing — so the trace is never ahead of the truth.
Seams The LLM is a swappable seam — real OpenRouter, scripted fake for deterministic tests, or rule-based demo needing zero keys. Same runtime everywhere.
